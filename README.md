# Job Feed — личный агрегатор вакансий

Что это: код сам, раз в день, забирает вакансии с RemoteOK, Remotive, Himalayas,
Working Nomads, а также с бордов Greenhouse и Lever выбранных тобой компаний
(см. `lib/companies.mjs`), отфильтровывает по твоим критериям
(см. `lib/criteria.mjs`) и сохраняет в базу. Отдельная страница показывает
результат с фильтрами.

После настройки (см. ниже) это не требует ни кода, ни ручных действий —
всё работает по расписанию само.

---

## Что нужно сделать один раз (~20 минут)

### Шаг 1. Создать бесплатный аккаунт на GitHub
Если его ещё нет: https://github.com/signup

### Шаг 2. Создать репозиторий и загрузить туда этот код
1. На github.com нажми **New repository**, назови, например, `job-feed`, сделай **Private**.
2. Загрузи в него все файлы из этой папки (можно перетащить через веб-интерфейс
   GitHub — кнопка "Add file" → "Upload files" — либо через `git push`, если
   пользуешься git).

### Шаг 3. Создать бесплатный проект в Supabase (база данных)
1. Зайди на https://supabase.com → **Start your project** → зарегистрируйся.
2. Создай новый проект (**New project**), любое название, регион — ближайший.
3. Подожди 1-2 минуты, пока проект поднимется.
4. Слева меню → **SQL Editor** → **New query** → скопируй туда содержимое
   файла `supabase/schema.sql` из этой папки → нажми **Run**.
5. Слева меню → **Project Settings** → **API**. Тебе понадобятся:
   - **Project URL** (выглядит как `https://xxxxx.supabase.co`)
   - **anon public** ключ (длинная строка)
   - **service_role** ключ (тоже длинная строка, но эту НИКОМУ не показывай
     и не вставляй на сайт — только в GitHub Secrets, см. ниже)

### Шаг 4. Подключить Supabase к GitHub Actions (чтобы сбор работал сам)
1. В твоём репозитории на GitHub: **Settings** → **Secrets and variables** →
   **Actions** → **New repository secret**.
2. Добавь два секрета:
   - `SUPABASE_URL` = твой Project URL
   - `SUPABASE_SERVICE_ROLE_KEY` = твой service_role ключ
3. Всё, теперь workflow `.github/workflows/daily-fetch.yml` сможет писать в базу.
   Он запускается каждый день в 06:00 UTC автоматически, а также его можно
   запустить вручную: вкладка **Actions** → **Daily job fetch** → **Run workflow**
   (удобно, чтобы проверить сразу, не дожидаясь завтрашнего дня).

### Шаг 5. Вставить свои ключи в сайт
1. Открой файл `site/index.html` в этом репозитории (можно прямо на GitHub —
   нажми на файл → карандаш "Edit").
2. Найди строки:
   ```js
   const SUPABASE_URL = "YOUR_SUPABASE_URL";
   const SUPABASE_ANON_KEY = "YOUR_SUPABASE_ANON_KEY";
   ```
3. Замени на свой Project URL и **anon public** ключ (НЕ service_role —
   anon ключ безопасно показывать в браузере, для него в базе настроен
   доступ только на чтение).
4. Сохрани (Commit changes).

### Шаг 6. Включить GitHub Pages, чтобы страница была доступна по ссылке
1. В репозитории: **Settings** → **Pages**.
2. Source: **Deploy from a branch**. Branch: `main`, папка: `/site`.
   Если GitHub не даёт выбрать `/site` — переименуй папку `site` в `docs`
   и укажи `/docs` (тогда поправь и путь при загрузке в шаге 2, не критично).
3. Сохрани. Через минуту-две GitHub покажет ссылку вида
   `https://твой-логин.github.io/job-feed/` — это и есть твоя страница.

## Как этим пользоваться дальше

Ничего делать не нужно. Раз в день код сам:
1. Сходит на сайты с вакансиями (RemoteOK, Remotive, Himalayas, Working
   Nomads) и на борды Greenhouse/Lever компаний из `lib/companies.mjs`.
2. Отфильтрует по ролям/индустриям/опыту/таймзонам/зарплате из
   `lib/criteria.mjs`.
3. Сохранит новые вакансии в базу, пометит их как "новое".
4. На странице (ссылка из шага 6) ты увидишь обновлённый список — просто
   открывай её как закладку.

## Если захочешь поменять критерии

Открой `lib/criteria.mjs` прямо на GitHub, поправь списки ролей/индустрий/
исключений или `maxExperienceYears`, сохрани — при следующем автозапуске
(или ручном запуске через Actions → Run workflow) фильтр применится с
новыми критериями.

## Если захочешь добавить компании в Greenhouse/Lever

По умолчанию `lib/companies.mjs` пустой — Greenhouse и Lever не имеют
общего каталога вакансий, поэтому нужно явно вписать токены нужных
компаний (инструкция, как их найти, — в комментарии в самом файле).

## Ограничения, которые стоит держать в голове

- Фильтр по размеру компании (1–30 человек) не применяется автоматически —
  ни один из источников не отдаёт эти данные напрямую. Смотри на компанию
  вручную по ссылке, если это важно для конкретной вакансии.
- LinkedIn и Indeed сюда не входят (у них нет открытого API) — для них
  по-прежнему стоит держать отдельные сохранённые поиски / алерты на самих
  площадках.
- Если какой-то источник поменяет формат ответа, соответствующий блок в
  `lib/sources.mjs` может потребовать правки — остальные источники при этом
  продолжат работать (ошибка одного не ломает остальные).
